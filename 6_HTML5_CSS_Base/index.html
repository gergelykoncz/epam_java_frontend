HTML5 CSS Base notes

Cascading Style Sheets is a technology used to format HTML markup

  • Cascading (or inheritance) means that CSS rules can be inherited from the DOM parent nodes.
  • Every HTML element is addressable with a sophisticated enough CSS rule.
  • For making it easier, CSS supports classes with the .className notation.
  • It also supports DOM IDs, with the #id notation.

Syntax

A CSS rule consists of two parts, a selector and a declaration block.

    .myRule {
        font-weight: bold;
        color: red;
    }

Specificity

CSS rules have specificity. This means that the relevance of a rule is dependent on where it is defined. The specificity is as follows, from weakest to strongest:

  1. User agent default stylesheet
  2. Internal/external CSS
  3. Inline CSS (attached to specific DOM elements via HTML attributes)
  4. !important rules

Box model

Every DOM element is rendered as a rectangle. This rectangle has the following boundaries:

  1. The DOM content itself
  2. A padding
  3. A border
  4. A margin

Inline vs block elements

There are two main kinds of CSS elements:

Block elements
They have a line-break before and after, and take all horizontal space of their parent. A div is a block element for example. Their width and height can be controlled by the relevant CSS properties.
Inline elements
Inline elements are rendered in-line, without any line-break. Their size is dependent on their content. A span is a block element.
I'm a block div! I'm an inline span

These can be controlled by the display property. The three most important values are:

  • Block: displays an element as block.
  • Inline: displays an element as inline.
  • None: completely hides an element.

Positioning

We can position HTML elements with CSS. This can be controlled by the position property, and the left, right, top, bottom properties.

static
The element is positioned as it is in the DOM. This is the default value.
relative
The element is positioned relative to its DOM position.
absolute
The element is positioned relative its first parent which is not static.
absolute
The element is positioned relative to the viewport.
